Products API

Endpoints for managing and retrieving products in Mudirdokan. This API requires admin privileges for CRUD operations and category management.

1. Get Product by ID

Endpoint: /api/products/:id

Method: GET

Description: Retrieve detailed information about a specific product by its ID. This endpoint is restricted to Admins.

Request:

GET /api/products/1

Response:

{
  "product_id": 1,
  "product_name": "Apple",
  "product_price": 3.5,
  "product_description": "Fresh red apples",
  "product_category_id": 2
}

2. Create a New Product

Endpoint: /api/products

Method: POST

Description: Create a new product. Admins must provide product details such as name, price, description, and category.

Request:

POST /api/products
Content-Type: application/json

{
  "product_name": "Banana",
  "product_price": 1.2,
  "product_description": "Fresh ripe bananas",
  "product_category_id": 3
}

Response:

{
  "message": "Product created successfully",
  "product": {
    "product_id": 2,
    "product_name": "Banana",
    "product_price": 1.2,
    "product_description": "Fresh ripe bananas",
    "product_category_id": 3
  }
}

3. Update Product by ID

Endpoint: /api/products/:id

Method: PUT

Description: Update the details of an existing product. Admins can modify the product's name, price, description, and category.

Request:

PUT /api/products/2
Content-Type: application/json

{
  "product_name": "Yellow Banana",
  "product_price": 1.5,
  "product_description": "Fresh yellow bananas",
  "product_category_id": 3
}

Response:

{
  "message": "Product updated successfully",
  "product": {
    "product_id": 2,
    "product_name": "Yellow Banana",
    "product_price": 1.5,
    "product_description": "Fresh yellow bananas",
    "product_category_id": 3
  }
}

4. Delete Product by ID

Endpoint: /api/products/:id

Method: DELETE

Description: Delete a product by its ID. Only Admins are allowed to perform this operation.

Request:

DELETE /api/products/2

Response:

{
  "message": "Product deleted successfully"
}

5. Fetch All Categories

Endpoint: /api/products/category/all

Method: GET

Description: Retrieve a list of all product categories. Admins can access this endpoint to view the available categories.

Request:

GET /api/products/category/all

Response:

{
  "categories": [
    {
      "category_id": 1,
      "category_name": "Fruits"
    },
    {
      "category_id": 2,
      "category_name": "Vegetables"
    }
  ]
}

6. Fetch Products by Category

Endpoint: /api/products/category/:categoryId

Method: GET

Description: Retrieve a list of products belonging to a specific category. Admins can use this endpoint to fetch products by category.

Request:

GET /api/products/category/1

Response:

{
  "products": [
    {
      "product_id": 1,
      "product_name": "Apple",
      "product_price": 3.5,
      "product_description": "Fresh red apples",
      "product_category_id": 1
    },
    {
      "product_id": 2,
      "product_name": "Banana",
      "product_price": 1.2,
      "product_description": "Fresh ripe bananas",
      "product_category_id": 1
    }
  ]
}
